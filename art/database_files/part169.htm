APPLICATION DESIGN AND DEVELOPMENT‌

One of the key requirements of the relational model is that data values be atomic: mul- tivalued, composite, and other complex data types are disallowed by the core relational model. However, there are many applications where the constraints on data types im- posed by the relational model cause more problems than they solve. In Chapter 8, we discuss several complex data types, including semistructured data types that are widely used in building applications, object-based data, textual data, and spatial data.

Practically all use of databases occurs from within application programs. Corre- spondingly, almost all user interaction with databases is indirect, via application pro- grams. Database-backed applications are ubiquitous on the web as well as on mobile platforms. In Chapter 9, we study tools and technologies that are used to build appli- cations, focusing on interactive applications that use databases to store and retrieve data.
