Forgetting to Initialize Allocated Memory

With this error, you call malloc() properly, but forget to fill in some val- ues into your newly-allocated data type. Don’t do this! If you do forget, your program will eventually encounter an uninitialized read, where it


reads from the heap some data of unknown value. Who knows what might be in there? If you’re lucky, some value such that the program still works (e.g., zero). If you’re not lucky, something random and harmful.
