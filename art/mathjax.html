MathJax

What happen when you insert math formula in markdown.

There is no such thing as the LaTeX in markdown. Most of the markdown previewer use MathJax or katex, which is not LaTeX nor TeX, but is the tool to render some math formula which is written as if it were LaTeX, where LaTeX is just an abstraction, a way to express the formula but not rendering it. Things get annoying especially when people start saying something like "Hey, I use markdown plus LaTeX".

Since MathJax is not doing what LaTeX doing, several drawbacks of MathJax is concerned (If there isn't any, I will not give a shit about whether MathJax is LaTeX or not). Okay, MathJax is just not precise enough, and the default font is very ugly.

Employ MathJax in your website

Just read the document of MathJax. Note that MathJax version 2 support different fonts while version 3 does not. If you are willing to change the font, you should check the documentation of MathJax version 2.

Change the font in your website

Note that there is actually three ways to render your formula:

  1. CHTML
  2. HTML+CSS
  3. SVG

where CHTML only supports one font. You may change the configuration to HTML+CSS or SVG if you want to change the font. The configuration is determined based on your source. Anyway, you may read the source code on current website for a check. After you change the configuration, read the documentation of MathJax font to change the configuration of SVG or HTML+CSS to display your preferred font.

There are four kinds of fonts available.

  1. TeX, which is ugly under certain circumstances.
  2. STIX, which is ugly under most of the circumstances.
  3. Asana-Math, which is ugly under most of the circumstances, just like STIX.
  4. Neo-Euler, which is pretty cool because you rarely see it, and Knuth use this font in his book Concrete Mathematics
  5. Latin-Modern, which is preferred since it suits CMU serif font best, but looks not so good because of SVG rendering. Here is an exmaple of Latin-Modern and CMU Serif.
