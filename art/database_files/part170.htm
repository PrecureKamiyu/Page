Complex Data Types‌‌


The relational model is very widely used for data representation for a large number of application domains. One of the key requirements of the relational model is that data values be atomic: multivalued, composite, and other complex data types are dis- allowed by the core relational model. However, there are many applications where the constraints on data types imposed by the relational model cause more problems than they solve. In this chapter, we discuss several non-atomic data types that are widely used, including semi-structured data, object-based data, textual data, and spatial data.
