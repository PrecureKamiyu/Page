Org and Emacs

Org mode is very interesting. You can appoint the style sheet that will be used by the exported html file. To do this, you shall use the following code in your file

#+HTML_HEAD: <link rel = "stylesheet" type="text/css" href="style.css"/>
#+OPTIONS: html-style:nil

The first sentence tells Emacs to include the code in the head of the html file. The second sentence tells Emacs to disable the default style sheet.

And it is very simple to add yasnippets snippets since the plugin has already provided us with some useful keybindings to create snippets and try out snippets. And it is especially convenient compared to the Ultisnips in Vim. You can quickly insert a snippet and then go back to work with no pressure. The snippets source is automatically renewed (Okay, so is Ultisnips). Anyway, since that the installation of the plugins in Emacs is much easier (unless you are not in China), and since that the M-x can easily list numbers of the available commands and variables, I think Emacs is, in this scenario, more convient than Vim. Mostly because the plugin experience is much better (Of course, that is because of the Wall).

I don't know which editor is better, for real. One may think that there are a lot of useful features in Emacs, while one may think that Emacs is like a shit mountain, with a ton of useless things.
